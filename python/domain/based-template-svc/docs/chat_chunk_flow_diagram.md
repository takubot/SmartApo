## 🔍 プライベートファイルフィルタリングの詳細

### データベースクエリ例

```sql
-- プライベートファイル対応後のチャンク検索クエリ
SELECT chunk.*
FROM chunk
JOIN file_outline ON chunk.file_id = file_outline.file_id
WHERE chunk.file_id IN (1, 2, 3, 4, 5)  -- ボットに紐づくファイルID
  AND chunk.chunk_embedding IS NOT NULL
  AND (
    -- 共有ファイル（全員アクセス可能）
    file_outline.is_private = FALSE
    OR
    -- プライベートファイル（所有者のみアクセス可能）
    (
      file_outline.is_private = TRUE
      AND file_outline.owner_external_user_id = 'line_user_001'
    )
  )
ORDER BY similarity_score DESC
LIMIT 20;
```

## 📋 ボットの挙動の変化

### 変更点

1. **チャンク取得時のフィルタリング**
   - **変更前**: ボットに紐づくすべてのファイルのチャンクを検索対象
   - **変更後**: 共有ファイル + 現在のユーザーが所有者のプライベートファイルのみ検索対象

2. **ボット選択への影響**
   - **CHUNK_DISTRIBUTION方式**:
     - プライベートファイルのチャンクもカウント対象
     - ユーザーごとに異なるボットが選ばれる可能性がある
   - **DESCRIPTION_BASED方式**:
     - プライベートファイルの存在は直接考慮されない
     - ただし、プライベートファイルのチャンクがプロンプトに含まれるため、間接的に影響

3. **回答の内容**
   - **変更前**: グループ全体で共有されている情報のみを基に回答
   - **変更後**: 共有情報 + そのユーザー専用のプライベート情報を基に回答
   - 例: 「あなたの契約書によると...」のような個別化された回答が可能

### セキュリティ保証

- ✅ プライベートファイルのチャンクは、所有者以外のユーザーには一切見えない
- ✅ データベースレベルでのフィルタリングにより、誤って他人のプライベート情報が漏洩しない
- ✅ ボット選択やプロンプト構築の段階では、既にフィルタリング済みのチャンクのみが使用される

| ステップ         | 変更前             | 変更後                                    |
| ---------------- | ------------------ | ----------------------------------------- |
