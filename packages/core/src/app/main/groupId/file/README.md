# ファイル管理ページ (File Page) ドキュメント

## 概要

このフォルダには、DOPPELプラットフォームのファイル管理機能に関するすべてのコンポーネント、フック、テンプレートが格納されています。ユーザーがアップロードしたドキュメントの管理、検索、チャンクデータの閲覧などの機能を提供します。

## アーキテクチャ

### フォルダ構成

```
file/
├── page.tsx                    # Next.js 15のページエントリーポイント
├── ChunkDataTemplate/          # メインテンプレート
│   └── template.tsx           # 全体のUI統合とモーダル管理
├── components/                 # UIコンポーネント
│   ├── FileListView.tsx       # ファイル一覧表示
│   ├── fileCard.tsx           # ファイルカード
│   ├── ChunkDataView.tsx      # チャンクデータ表示
│   ├── ChunkSidePanel.tsx     # チャンク編集サイドパネル
│   ├── PerfectFileUploadModal.tsx  # ファイルアップロードモーダル
│   ├── BulkCategoryModal.tsx       # 一括カテゴリ設定モーダル
│   ├── TableView.tsx               # テーブル表示
│   └── chunkCard.tsx               # チャンクカード
└── hooks/                      # カスタムフック
    ├── useTemplate.ts          # 統合データ・UI状態管理フック
```

## 主要コンポーネント

### 1. ChunkDataTemplate (template.tsx)

- **役割**: アプリケーション全体のUI統合、モーダル管理、イベントハンドリング
- **特徴**:
  - ファイル一覧とチャンク一覧の表示制御
  - 確認モーダルの統一管理
  - エラーハンドリングと通知機能
  - 各機能の協調動作

### 2. FileListView

- **役割**: ファイル一覧の表示、検索、フィルタリング
- **機能**:
  - カード/リスト表示切り替え
  - ファイル名・内容での検索
  - カテゴリフィルタリング
  - 複数選択・一括操作
  - ボットアイコン表示

### 3. ChunkDataView

- **役割**: 選択されたファイルのチャンクデータ表示
- **機能**:
  - チャンク一覧表示
  - テーブルビュー（構造化データ用）
  - チャンク検索・編集
  - カテゴリ一括設定

## カスタムフック

### useTemplate

**中心的なデータ・UI状態統合フック**

- 全てのデータ取得（ファイル、チャンク、カテゴリ）を統合
- UI状態（検索、フィルタ、選択状態、モーダル）を一括管理
- ページネーション管理
- 各種操作（削除、更新、一括操作等）のイベントハンドラーを提供

## データフロー

```
1. useTemplate
   ├── SWR (ファイル一覧・チャンクデータ・カテゴリ取得)
   └── useState/useCallback (UI状態・イベント管理)

2. template.tsx
   ├── データフックから状態を受け取り
   ├── 各Viewコンポーネントに props として渡す
   └── モーダル・確認ダイアログを管理

3. 各Viewコンポーネント
   ├── propsとして受け取ったデータを表示
   ├── ユーザーイベントをハンドラーに渡す
   └── 子コンポーネント（Card等）にデータを配布
```

## 機能

### ファイル管理

- **アップロード**: PDF、Excel、Word、CSV等対応
- **プレビュー**: PDF/テキストファイルの新しいタブでのプレビュー
- **ダウンロード**: 元ファイルのダウンロード
- **削除**: 単体・複数削除（関連チャンクも同時削除）
- **検索**: ファイル名検索、コンテンツ内検索

### チャンクデータ管理

- **表示**: アップロードされたファイルから生成されたチャンク
- **編集**: チャンクタイトル・内容の編集
- **カテゴリ**: チャンクへのカテゴリ付与・管理
- **検索**: チャンク内容での検索

### ボット連携

- **アイコン表示**: ファイルに関連付けられたボットのアイコン表示
- **関連付け**: どのボットがどのファイルを使用しているかの表示

## 技術仕様

### フレームワーク・ライブラリ

- **Next.js 15**: React Server Components対応
- **React 19**: 最新のReact機能
- **TypeScript**: 型安全性
- **TailwindCSS**: スタイリング
- **HeroUI**: UIコンポーネントライブラリ
- **SWR**: データフェッチング・キャッシュ

### API統合

- **backend-first開発**: FastAPIエンドポイントからクライアント自動生成
- **生成されたAPIクライアント**: `@repo/api-contracts/based_template/service`
- **認証**: Firebase Authentication
- **マルチテナント**: グループベースのデータアクセス

### 状態管理

- **SWR**: サーバーサイド状態管理
- **React Context**: グローバル状態（グループ情報等）
- **カスタムフック**: ローカル状態管理

## 使用方法

### 基本的な流れ

1. **ファイルアップロード**: 「アップロード」ボタンからファイルを選択
2. **ファイル一覧**: アップロードされたファイルがカード形式で表示
3. **ファイル選択**: ファイルをクリックしてチャンクデータを表示
4. **チャンク管理**: チャンクの編集、カテゴリ設定、検索

### 検索機能

- **ファイル名検索**: デフォルトの検索モード
- **コンテンツ検索**: 「ファイル名と内容」を選択してファイル内容も検索対象に

### フィルタリング

- **カテゴリフィルタ**: 複数カテゴリでのフィルタリング
- **並び替え**: 作成日、ファイル名、拡張子での並び替え

## カスタマイズ・拡張

### 新しいファイルタイプの追加

`fileCard.tsx`の`getFileIcon`関数に新しい拡張子を追加:

```typescript
case "新しい拡張子":
  return <NewIcon className="text-color w-6 h-6" />;
```

### 新しいAPIエンドポイントの追加

1. バックエンドでエンドポイントを追加
2. `npm run gen:openapi:all`でクライアント生成
3. `useTemplate`フックで新しいAPIを使用

### UIコンポーネントの追加

1. `components/`フォルダに新しいコンポーネントを作成
2. `template.tsx`で統合

## トラブルシューティング

### ボットアイコンが表示されない

- `useTemplate`内のボット関連ロジックを確認
- APIエンドポイントが正しく実装されているか確認
- ネットワークタブでAPI呼び出しを確認

### ファイルアップロードが失敗する

- ファイルサイズ制限を確認
- サポートされているファイル形式か確認
- バックエンドのエラーログを確認

### 検索が動作しない

- デバウンス処理が正しく動作しているか確認
- API呼び出しのパラメータを確認
- 検索範囲設定（ファイル名のみ/内容含む）を確認
